🌚 ponder-2: Tell the Truth: Design by Contract and Styles Matter

If your design contract misses precondition, your program will crash.
check answer

False.
The precondition tells the user the correct input format. If the user does not know it, and input the inappropriate parameter, the programme probably can crash, or probably can still run but output incorrectly, or probably will throw out an exception. No one knows what will happen.
If the code snippet is a small part of a huge project and it does not crash, rather than, it pass on incorrect output to its caller function, this will result in much more bigger trouble for debugging.
Hence, specify the precondition is crucial important in the design contract.

🌚 ponder-6: Comparison between if-else and case-switch

description switch | nested if
It is easy to use when there are multiple choices and easy to understand.
It uses multiple expressons for multiple choices.
It checks only constant values, i.e. numeric values, character expressions and boolean value
It can check both constant values and variables or relational expressions
It can check a range of values.
    If the given value falls in the specified range,
    then the statements under it will be executed.

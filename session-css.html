<

The child element will naturally inherit a CSS property with its value from the parent element if the CSS property is not specified.
hence, if some contents is a sub-part of another content, you can nest their block tags to unify their styles without using class and id.

Inheritance works through multiple generations as well. If a tag like the <em> or <strong> tag appears inside of a <p> tag, then the <em> and the <strong> tags also inherit properties from any style applied to the <body> tag.

inheritance is a really big timesaver. Imagine if no properties were passed onto nested tags and you had a paragraph that contained other tags like a <strong> tag, an <em> tag to emphasize text and an <a> tag to add a link. If you created a style that made the paragraph text white and 32 pixels tall, using the Varela Round font, it would be weird if all the text inside the <em>, <strong> and <a> tags reverted to its regular, “browser boring” style. You’d then have to create another style to format the <em> tag to match the appearance of the <p> tag. What a drag.

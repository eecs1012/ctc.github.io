🌚 ponder02: Edge Case

The situation where the test examines either the beginning or the end of a range, but not the middle, is called an edge case.

Qualitatively different behavior happens at boundaries. As such, they tend to have special code dedicated to them in the implementation.

Example: An Oven's User Interface

Setting temperature and time are two basic functions: assume the temperature range is 300F to 450F, and the maximum time slot be 12 hours. What test cases should be designed?

Example: "SmartBalance" Mode from a Stupid Thermostat

I bought a Thermostat to control the house's temperature. On Christmas, suddenly, the furnace stopped and the airconditioner started, and even if I turned off the air-conditioner and turned on the furnace, this cooling process started again. Why is this happending? Then I discovered there is a "SmartBalance" mode. If the housing temperature is above 24C, then the airconditioner will be turned on automatically, and if the housing temperature is below 18C, the furnace will be turned on automatically.

Q1:To avoid this off-season error happen, what condition should be added to turn on this mode?
Q2:What test cases should be designed?

solution 1: the outside temperature can be taken into consideration. For example, if the outdoor temperature is below 20C and the indoor temperatur is above 24C, then the mode should be turned off.

solution 2: add an attribute "season" or "month" into this mode. If the season is winter, then the airconditioner should not be turned on.If it is summer, then the furnace should always be turned off.

🌚 ponder01: Short Circuit Execution

Short Circuit is a programming concept related to compilers. In a compound conditional expression, the second argument is evaluated only if the first argument does not suffice to determine the value of the whole expression:

a    b    a and b can a determine a and b ?
true true true
true false false
false true false
false false false

Can you code this table into a nested-if statement?


  if(a == true){
    if(b == true){
      // a && b <-- true
    }
  }
  else{
	  // a && b <-- false
  }

a b a or b can a determine a and b ?
true true true
true false true
false true true
false false false

Can you code this table into a nested-if statement?


  if(a == false){
    if(b == false){
      // a || b <-- false
    }
  }else{
	 //  a || b <-- true
  }

When the first argument of the AND function evaluates to false, the overall value must be false;
and when the first argument of the OR function evaluates to true, the overall value must be true.

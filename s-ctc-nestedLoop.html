🌚 ponder-1: Nested For Loop: Relationship between the Inner Loop and the Outer Loop

When you "nest" two loops, the outer loop takes control of the number of complete repetitions of the inner loop.
When working with nested loops, the outer loop changes only after the inner loop is completely finished (or is interrupted.).

Postcondition: Draw a rectangle consists of #. The rectangle has "row" number of rows, and "col" number of columns.
i,j and k are the standard counter variables. By using them you imply the variables are used to keep loop count and nothing else. However, as a newbie computational thinker, very often you need to visualize your code in a better understanding way. In the example above, "row" means in total how many rows are to be drawn, "col" means the number of columns each row has(the number of # each row has). If we view the final output as a picture in a 2D plane, then x, y, (and z) can also be useful. They can help you to recall the scene when do the work in 2D or 3D coordinates.
